# Снэплот · демо витрины цифровых товаров

[![License: MIT](https://img.shields.io/badge/License-MIT-22d3ee.svg)](LICENSE)
[![Static site](https://img.shields.io/badge/stack-static%20HTML%2FCSS%2FJS-0a0c13?logo=html5&logoColor=fff)](https://pages.github.com/)
[![Demo](https://img.shields.io/badge/режим-локальное%20демо-5eead4.svg)](#-демо-вход)

**Снэплот** — открытый учебный/презентационный проект: лендинг, каталог, карточка товара, корзина, оформление заказа, личный кабинет, демо-выдача после покупки и админка. Бэкенда нет: данные и «API» эмулируются в браузере (`localStorage` + маршрутизатор в `assets/js/core/api.js`).

Подходит для портфолио, защиты проекта или как основа для статического фронтенда.

---

## Возможности

- Официальный магазин (один продавец), эскроу и правила — в единой логике демо
- Каталог с фильтрами, избранное, корзина, быстрый просмотр, сравнение
- Регистрация и вход (локальные учётные записи в браузере)
- Заказы, кошелёк, бонусные баллы (демо)
- Страница выдачи после оплаты (анимация проверки, копирование полей)
- Админка: добавление позиций (для роли администратора)
- PWA-заготовка (`sw.js`), тёмная визуальная тема

## Быстрый старт

Клонируйте репозиторий и поднимите **любой статический HTTP-сервер** из корня проекта (открытие через `file://` может ломать загрузку модулей и JSON).

```bash
git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git
cd YOUR_REPO
```

```bash
npx --yes serve .
```

или

```bash
python -m http.server 8080
```

Откройте в браузере адрес, который вывел сервер (например `http://localhost:3000` или `http://localhost:8080`).

## Демо-вход

| Роль    | Email               | Пароль |
|---------|---------------------|--------|
| Админ   | `admin@demo.snaplot` | `demo` |

Учётка администратора создаётся/нормализуется при первой загрузке страницы с подключённым `api.js` в демо-режиме (мета `snaplot-demo`).

Обычные пользователи регистрируются на странице регистрации — данные хранятся локально в браузере.

## Публикация на GitHub Pages

1. В репозитории: **Settings → Pages**.
2. **Build and deployment**: Source — *Deploy from a branch*.
3. Branch: `main` (или `master`), folder: **`/` (root)**.
4. После деплоя сайт будет доступен по адресу вида  
   `https://YOUR_USERNAME.github.io/YOUR_REPO/`

Главная страница — `index.html` в корне; пути к `pages/` и `assets/` уже относительные.
